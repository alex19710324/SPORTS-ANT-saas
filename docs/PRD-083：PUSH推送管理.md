# PRD-083：PUSH推送管理（适配Trae开发）

## 1. 概述
- **目标用户**：运营人员、开发人员。
- **核心价值**：提供统一的APP/小程序PUSH推送管理平台，支持多通道（极光、个推、腾讯TPNS）、消息模板、定时推送、用户分群、效果统计。
- **技术目标**：基于Spring Boot，集成多家PUSH服务商，支持通知栏消息、透传消息、富媒体消息。

## 2. 用户故事
- 作为**运营**，我想创建一条活动推送，选择“近7天活跃会员”分群，设置今晚8点定时发送，并实时查看点击率。

## 3. 功能列表（按优先级）

### 3.1 推送配置（P0）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
|----|----------|------|--------------|----------|
| PS01 | 通道配置 | 配置极光/个推/腾讯TPNS等通道的AppKey、Secret。 | 接口：`/api/push/channels` | 配置生效 |
| PS02 | 模板管理 | 创建推送消息模板，支持变量替换。 | 接口：`/api/push/templates` | 模板可用 |

### 3.2 推送任务（P0）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
|----|----------|------|--------------|----------|
| PS03 | 任务创建 | 选择模板、目标人群（全部/分群）、推送时间、跳转链接。 | 接口：`POST /api/push/tasks` | 创建成功 |
| PS04 | 定时推送 | 支持立即发送或定时发送。 | 定时任务 | 准时发送 |
| PS05 | 推送记录 | 查看历史推送任务状态、送达数、点击数。 | 接口：`GET /api/push/history` | 数据准确 |

### 3.3 效果统计（P1）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
|----|----------|------|--------------|----------|
| PS06 | 送达/点击统计 | 实时统计推送任务的送达率、点击率。 | 数据中台 | 实时更新 |

## 4. 依赖模块

- 用户分群（PRD-082）
