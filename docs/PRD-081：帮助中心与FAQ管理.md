# PRD-081：帮助中心与FAQ管理（适配Trae开发）

## 1. 概述
- **目标用户**：顾客、员工、客服。
- **核心价值**：构建统一的帮助中心，集中管理常见问题、操作指南、视频教程，支持多语言、全文检索，提升自助服务率。
- **技术目标**：基于Vue3 + Elasticsearch，提供分类导航、关键词搜索、阅读量统计、评价反馈。

## 2. 用户故事
- 作为**顾客**，我在小程序中点击“帮助中心”，搜索“如何退会员卡”，找到图文指南，问题解决。
- 作为**客服**，我转发帮助中心文章给顾客，减少重复解释。

## 3. 功能列表（按优先级）

### 3.1 内容管理（P0）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
|----|----------|------|--------------|----------|
| HC01 | 分类管理 | 创建多级分类（会员、支付、设备、活动）。 | 接口：`/api/help/categories` | 分类清晰 |
| HC02 | 文章管理 | 创建/编辑帮助文章，支持富文本、图片、视频链接，设置关联标签。 | 接口：`/api/help/articles` | 编辑便捷 |
| HC03 | 多语言支持 | 文章可配置多语言版本，根据用户语言自动展示。 | 语言底座 | 展示准确 |

### 3.2 用户端展示（P0）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
|----|----------|------|--------------|----------|
| HC04 | 帮助中心首页 | 展示热门问题、分类导航、搜索框。 | 接口：`GET /api/help/home` | 加载≤2秒 |
| HC05 | 全文检索 | 支持关键词搜索，高亮显示匹配内容。 | Elasticsearch | 响应≤1秒 |
| HC06 | 阅读反馈 | 文章底部提供“有帮助/无帮助”评价，记录统计。 | 接口：`POST /api/help/feedback` | 反馈收集 |

### 3.3 统计分析（P2）

|----|----------|------|--------------|----------|
| HC07 | 阅读排行 | 统计文章阅读量、有帮助率，指导内容优化。 | 数据中台 | 报表清晰 |
