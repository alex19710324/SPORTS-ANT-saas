# PRD-017：人事行政与绩效管理（适配Trae开发）

## 1. 概述
- **目标用户**：HR人事专员、店长、部门经理、员工本人、财务人员（薪酬核算）。
- **核心价值**：实现员工全生命周期数字化管理，从招聘入职、考勤排班、绩效考核、薪酬核算到培训发展，构建透明、高效、合规的人事行政体系，提升组织效能。
- **技术目标**：基于Spring Boot + Vue3微服务架构，集成工作流引擎、规则引擎、财务底座（薪酬）、通讯社媒底座（通知、招聘发布），支持多区域劳动法合规，数据实时同步。

## 2. 用户故事
- 作为**HR专员**，我想要发布招聘职位，筛选简历，安排面试，发送录用通知，并管理员工档案（合同、资料），确保用工合规。
- 作为**店长**，我想要查看门店员工排班，审批请假、加班，进行月度绩效评估，了解团队人力成本。
- 作为**员工**，我想要在手机上打卡，查看我的考勤记录、工资条、绩效结果，申请请假，参与培训。
- 作为**财务人员**，我想要获取准确的考勤、绩效数据，用于薪酬计算。
- 作为**培训经理**，我想要创建培训计划，组织线上/线下培训，记录培训结果。

## 4. 技术架构
- **前端**：Vue3 + Element Plus（后台），嵌入员工工作台（移动端）。
- **后端**：Spring Boot微服务，独立HR模块。
- **数据**：MySQL（员工信息、考勤、绩效）、Redis（实时打卡）、Elasticsearch（简历检索）。
- **底座依赖**：
  - 通讯社媒底座：招聘发布、通知推送。
  - 财务底座：薪酬计算。
  - 工作流引擎：审批流程。

## 5. 接口设计示例
- **考勤打卡**：
  - `POST /api/attendance/clock`
  - 请求：
    ```json
    {
      "employeeId": 123,
      "type": "上班",
      "time": "2025-03-03T09:00:00Z",
      "location": { "lat": 31.23, "lng": 121.47 }
    }
    ```

- **绩效评估提交**：
  - `POST /api/performance/evaluate`
  - 请求：`{ "targetUserId": 456, "scores": { "KPI1": 90, "KPI2": 85 } }`
