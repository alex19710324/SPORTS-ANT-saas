# PRD-094：会员标签体系（细化CRM）（适配Trae开发）

## 1. 概述
- **目标用户**：会员运营、市场专员。
- **核心价值**：构建精细化会员标签体系，支持手动/自动打标签，用于精准营销和个性化服务。
- **技术目标**：基于规则引擎 + AI，支持静态标签（如性别）和动态标签（如高消费、易流失）。

## 2. 用户故事
- 作为**运营**，我创建“沉睡会员”标签（近90天未消费），圈选人群发送唤醒优惠券。

## 3. 功能列表（按优先级）

### 3.1 标签管理（P0）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
|----|----------|------|--------------|----------|
| TL01 | 标签定义 | 创建标签（名称、类型、规则）。 | 接口：`/api/tags` | 创建成功 |
| TL02 | 标签分类 | 支持分类（基础属性、消费行为、互动行为）。 | 分类 | 清晰 |

### 3.2 自动打标签（P0）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
|----|----------|------|--------------|----------|
| TL03 | 规则打标 | 基于消费、行为等规则自动给用户打标签。 | 规则引擎 | 准确 |
| TL04 | AI打标 | AI自动生成潜力标签（如“高价值”“易流失”）。 | AI底座 | 准确率≥80% |

### 3.3 标签应用（P1）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
