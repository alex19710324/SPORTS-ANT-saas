# PRD-059：会员日活动管理（适配Trae开发）

## 1. 概述
- **目标用户**：会员运营、市场专员、门店店长。
- **核心价值**：打造周期性会员日营销活动，通过专属权益、积分加倍、限时秒杀、会员专属价等形式，提升会员活跃度与复购率。
- **技术目标**：基于活动引擎+会员系统，支持灵活配置会员日规则、自动化执行、效果追踪，会员日复购率提升目标≥20%。

## 2. 用户故事
- 作为**会员运营**，我设定每月8号为会员日，设置当日会员消费双倍积分、指定商品8折、积分兑换8折优惠。
- 作为**会员**，我在会员日收到小程序推送，进入活动页面享受专属权益，下单购买。
- 作为**店长**，会员日结束后，我查看活动期间会员消费占比、新会员转化数据。

## 3. 功能列表（按优先级）

### 3.1 会员日规则配置（P0）

| ID | 功能名称 | 描述 | 技术实现要点 | 验收标准 |
|----|----------|------|--------------|----------|
| MD01 | 日期配置 | 支持固定日期（如每月8号）、动态日期（如每月第二个周六）。 | 规则引擎 | 配置灵活 |
| MD02 | 权益配置 | 设置会员日专属权益：积分加倍、指定折扣、专享商品、兑换优惠。 | 接口：`POST /api/memberday/rules` | 权益生效 |
| MD03 | 适用人群 | 可指定全员或特定等级会员参与。 | 人群筛选 | 准确应用 |
